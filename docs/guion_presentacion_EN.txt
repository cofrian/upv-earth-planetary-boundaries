3 · METHODOLOGY  (target 5:00 total)
================================================================================

--------------------------------------------------------------------------------
3.1 — Transformers  (target 1:30)
--------------------------------------------------------------------------------

[Slide: Transformers — four cards + encoding diagram.]

"We started with the cheapest, most elegant option: pre-trained
embeddings, NO fine-tuning.

The intuition: if a model already understands scientific English, we
don't need to retrain it — we just need to ask the right geometric
question.

Four encoders, in increasing order of domain specificity. BERT — the
general-language baseline. RoBERTa — same family, trained harder.
SciBERT — pre-trained on a scientific corpus. And SPECTER2, the special
one: trained specifically on SCIENTIFIC PAPERS, using their citation
graph. It doesn't just understand science — it understands HOW PAPERS
RELATE to each other.

[Point to the diagram.]

The pipeline is simple — and that's the point. Encode the abstract,
encode the nine boundary definitions, take cosine similarity, assign
the boundary with the highest score.

One encoding per paper, nine cheap comparisons, fully explainable.
That gave us the hard baseline the LLMs had to beat."

[Transition.] "But embeddings have a ceiling. When a paper is genuinely
ambiguous, we need something that can REASON."

--------------------------------------------------------------------------------
3.2 — LLM Models & Prompt Engineering  (target 1:30)
--------------------------------------------------------------------------------

[Slide: LLM Models — three models, four prompts.]

"Two questions in parallel: which model, and how do we prompt it.

Three candidates. Llama 3.1 8B — fast but too shallow. Gemma 4 26B —
fluent, but we'd ask for XML and it'd write us poetry. And our winner:
QWEN 2.5 14B — the best trade-off between reasoning quality,
instruction-following, and disciplined structured output.

[Move to prompts.]

And four prompt versions — because the prompt is a piece of ENGINEERING,
not an afterthought.

v1 Few-Shot — brittle outside the example domains.
v2 Chain-of-Thought — recall up, over-confidence up.
v3 — v2 plus a background filter that discards the generic intro.
v4 Principle-Driven XML — we give the model the explicit scientific
PRINCIPLES behind each boundary and force a strict XML output. This is
what we ship.

[Pause.] "The headline: Qwen 2.5 14B + the v4 prompt. Hold those two
names."

--------------------------------------------------------------------------------
3.3 — The Agent Cascade  (target 1:30)
--------------------------------------------------------------------------------

[Slide: Agents — five-box pipeline.]

"Calling a 14B model on EVERY paper in a 50,000-paper corpus is
expensive. So we don't. We built a CASCADE where lightweight agents
filter, and the heavy ones only fire on the hard cases.

AGENT 1, the Extractor — a Qwen 2.5 3B. Pulls structured signals from
the title and abstract.

In parallel, the SCORER runs a deterministic keyword overlap against
our reference matrix. No LLM call.

Both feed the ROUTER, which asks a single question: do they agree this
paper is irrelevant? If yes — FAST-SKIP. No expensive call. This is
where the savings live.

If it's ambiguous, it goes to AGENT 3 — the Judge — Qwen 2.5 14B with
the v4 prompt. And only in the dangerous quadrant — Judge says 'none'
but the scorer disagrees — does AGENT 4, the CRITIC, fire as a second
opinion.

[Beat.] "The key insight is ECONOMIC: only the genuinely ambiguous
papers reach the expensive model. Same accuracy, a fraction of the
compute."

--------------------------------------------------------------------------------
3.4 — Evaluation Metrics  (target 0:30)
--------------------------------------------------------------------------------

[Slide: accuracy metrics + bias slide.]

"In a multi-label problem, accuracy is not one number. We use six
precision metrics — Strict Top-1, Relaxed Top-1, Hit@K, Any Overlap,
Exact Match and Jaccard — plus three behavioural ones: True Negative
Rate, Positivity Bias and False Negatives.

Accuracy plus bias plus behaviour. Full picture, not one score."

4.2 — Global Performance Comparison  (target 1:00)
--------------------------------------------------------------------------------

[Slide: bar chart.]

"The headline chart. The magenta bars are v4.

Against v1: Strict Top-1 goes UP plus 13.6 points to 72.1%. Any Overlap

And False Negatives — the metric we cared about most — drops 26.8

[Point at TNR.] "And critically: True Negative Rate stays at 76%. v2
chased recall and COLLAPSED to 54% — it was inventing labels. v4 buys
back the recall WITHOUT losing bias control."

6 · CONCLUSIONS & LIMITATIONS  (target 1:30)
================================================================================

[Slide: two columns.]

"Three conclusions.

ONE — prompting is REASONING DESIGN. It's not about magic templates;
it's about aligning instructions with the model's internal logic.

TWO — behavioural divergence is real. Same model, different prompt,
DRASTICALLY DIFFERENT behaviour. We measured it.

THREE — the winning formula is principle-driven. Giving the model the
scientific principles balances RIGOUR — high True Negatives — with
COVERAGE — high recall.

[Move to limitations.] And being honest with the limits:

- Few validated papers — limits statistical power.
- That's why we couldn't responsibly fine-tune: ceiling left on the
  table.
- Rare classes — PB5 and PB6 — low sample sizes, directionally correct
  but not statistically tight.

These aren't excuses. They're the NEXT piece of work."
